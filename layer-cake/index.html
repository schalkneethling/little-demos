CSS cascade layers

The @layer cake

Drag to restack the layers. The layer on top wins the cascade, so it gets to ice the cake. Every layer still contributes the parts no other layer touches, so reordering changes only the one property they all compete for: the icing colour.

The result

The icing colour and the flavour word both follow the winner.

The flavour label, set in tracked capitals.

Both contested values come straight from the cascade: the icing colour, and the flavour word, which each layer supplies through the content property on a pseudo-element. This is for demonstration only. Meaningful text set through content cannot be selected, is not found by in-page search, is invisible to translation tools, and reaches screen readers inconsistently across browser and reader pairings. In real work the word would live in the markup; here the announcement is carried by a live region instead.
